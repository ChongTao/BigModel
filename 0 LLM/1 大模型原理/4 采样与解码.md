# 一 采样与解码是什么

大语言模型在生成文本时，并不是“直接输出完整答案”，而是每一步都先算出“下一个 token 的分数（logits）”，再转成概率分布，然后根据某种策略选出一个 token，继续往后生成。

这个“如何从候选 token 中做选择”的过程，就是解码（decoding）。

要注意：

- **解码**是更大的概念，指“最后到底怎么选 token”
- **采样**通常指其中带随机性的那一类选择方式

例如：

- greedy decoding：不是采样
- beam search：通常也不叫采样
- top-k / top-p：更典型地属于采样策略

## 1.1 为什么需要解码策略

模型输出的是一组概率，不是唯一答案。例如某一步可能是：

- `北京`：0.42
- `上海`：0.33
- `广州`：0.08

系统必须决定：

- 是永远选概率最大的？
- 还是允许一定随机性？
- 是偏稳定，还是偏多样？

这就是采样与解码要解决的问题。

如果和前面的 Transformer 内容连起来，可以把生成链路简化理解成：

`hidden states -> vocab logits -> 概率分布 -> 解码策略 -> 输出 token`

## 1.2 常见解码方式

### 1.2.1 Greedy Decoding

每一步都选当前概率最高的 token。

优点：

- 最稳定。
- 最容易复现。

缺点：

- 容易保守。
- 容易陷入重复或局部最优。

适用：

- 信息抽取
- 结构化输出
- 高确定性问答

### 1.2.2 Beam Search

不是只保留一条生成路径，而是同时保留多条高分路径，最后再选整体得分更优的序列。

优点：

- 比 greedy 更有全局搜索能力。

缺点：

- 计算更重。
- 在开放文本生成里不一定总是更自然。
- 在现代通用对话型 LLM 中，也不像 Top-p 那样常用。

适用：

- 翻译
- 摘要
- 偏序列优化任务

### 1.2.3 Top-K Sampling

每一步先保留概率最高的 K 个 token，再在这 K 个里随机采样。

优点：

- 保留一定多样性。
- 避免从极低概率 token 中乱选。

缺点：

- K 固定时，对不同分布适应性一般。

### 1.2.4 Top-p Sampling

也叫 nucleus sampling。不是固定保留 K 个 token，而是保留累计概率达到 `p` 的最小候选集合。

优点：

- 更自适应。
- 生成效果通常比固定 Top-K 更自然。

缺点：

- 参数理解门槛比 Top-K 略高。

## 1.3 Temperature 的作用

Temperature 会调整 logits / 概率分布的尖锐程度，再影响后续如何选择 token：

- 温度低：更保守，更稳定。
- 温度高：更多样，更发散。

可以把它理解为：

- 低温：把高概率 token 进一步拉高，让分布更尖
- 高温：把概率分布拉平，让更多候选有机会被选中

它通常和 Top-K / Top-p 一起使用。

很多实现里：

- `temperature = 0` 往往近似退化为 greedy 或极低随机性输出
- `temperature` 本身不是单独的“解码算法”，而是对分布做调节的控制参数

## 1.4 不同任务的常见配置倾向

- 代码、抽取、工具调用：低温，必要时配合较小的 top-p，偏保守。
- 常规问答、总结：中低温，可配合适中的 top-p。
- 创意写作、文案生成：中高温，通常更依赖 top-p / top-k 提供多样性。

## 1.5 采样不决定知识正确性

要特别区分：

- 采样策略决定“怎么选”。
- 模型知识决定“候选里有没有正确答案”。

如果模型本身不知道某个事实，把温度调到 0，也只是更稳定地输出一个可能错误的答案。
