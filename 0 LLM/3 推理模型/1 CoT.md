# 一 Chain-of-Thought（CoT）

Chain-of-Thought（CoT，思维链）是一种让模型**显式写出推理过程**的方式。它的核心不是只要最终答案，而是让模型把中间思路也展开出来。

可以把它理解为：

- 普通回答：直接给结论
- CoT：先把思考步骤展开，再给结论

## 1.1 CoT 解决什么问题

对于简单问题，模型往往可以直接回答；但在更复杂的任务中，一步直答容易出错。

CoT 主要解决的是：

- 数学题
- 逻辑推理
- 多步分析
- 需要分解过程的问题

它的作用，不是给模型新增外部能力，而是让模型更好地利用已有推理能力。

## 1.2 CoT 的基本思路

CoT 的典型做法，是在提示中鼓励模型逐步思考，例如：

```text
Let's think step by step.
```

或者更明确地要求：

- 请分步骤推理
- 请先分析，再给最终答案
- 请写出中间判断过程

这样做的本质，是把模型从“直接输出答案”切换到“先生成推理轨迹，再收敛到答案”。

## 1.3 优点

- 对复杂任务通常更稳
- 推理过程更可解释
- 容易和 few-shot 示例结合

## 1.4 局限

CoT 也有明显边界：

- 它只有“思考”，没有“行动”
- 当任务需要外部信息时，仍然容易产生幻觉
- 推理链一旦前面出错，后面可能“错得很自洽”

这也是为什么后面会进一步发展出：

- ToT：不只走一条推理链，而是探索多条分支
- ReAct：不只思考，还能行动
- Reflexion：在生成后再回头反思与修正

## 1.5 一句话总结

CoT 的核心价值是：**让模型把隐含推理过程显式展开，从而提升复杂问题的求解能力。**
