# 一 Self-Ask

Self-Ask 是一种“先拆问题，再逐步求解”的推理方式。它的核心思路是：模型先把原问题改写成若干子问题，再逐个回答，最后汇总结果。

## 1.1 Self-Ask 解决什么问题

很多复杂问题并不是因为单步推理太难，而是因为问题本身包含多个隐藏子问题。

Self-Ask 适合：

- 多跳问答
- 文档分析
- 知识整合
- 需要任务拆解的问题

## 1.2 核心思路

典型形式是：

```text
Q: ...
Let's break this into sub-questions.
1. ...
2. ...
```

也就是说，模型不急着直接回答，而是先问自己：

- 这个问题要拆成哪几步？
- 哪个子问题应该先回答？
- 子问题之间是什么关系？

## 1.3 优点

- 推理结构更清晰
- 更适合多跳问题
- 比直接长答案更容易控制

## 1.4 局限

- 不包含行动能力
- 对提示约束较敏感
- 如果拆分本身错了，后续步骤也会一起偏掉

## 1.5 一句话总结

Self-Ask 的核心价值是：**把复杂问题先拆成子问题，再逐步回答，从而提高推理的结构化程度。**
