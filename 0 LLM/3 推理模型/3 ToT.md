# 一 Tree-of-Thoughts（ToT）

Tree-of-Thoughts（ToT，思维树）可以看作 CoT 的扩展：模型不再只走一条线性思路，而是同时探索多个推理分支，再从中挑选更优路径。

## 1.1 ToT 和 CoT 的区别

可以简单对比：

- CoT：一条思路往下走
- ToT：生成多个候选思路，再搜索、比较和筛选

所以 ToT 更像是在把“推理”变成一种带搜索的过程。

## 1.2 ToT 解决什么问题

ToT 适合那些：

- 一条思路不一定够
- 需要探索多种可能路径
- 最优解不容易一步想到

典型场景包括：

- 高复杂度推理
- 规划问题
- 谜题与优化问题

## 1.3 ToT 的核心机制

ToT 的典型过程通常包括：

1. 先生成多个候选 thought
2. 对这些 thought 做评估或打分
3. 保留更有前景的分支
4. 继续向下扩展，直到找到较优解

实现上常会用到：

- DFS
- BFS
- Beam Search

## 1.4 优点

- 探索能力比 CoT 更强
- 对多路径问题更有优势
- 更适合“需要搜索而不是只需要线性推导”的任务

## 1.5 局限

- 计算成本明显更高
- 搜索策略设计更复杂
- 如果分支评估不准，搜索效果会变差

## 1.6 一句话总结

ToT 的核心价值是：**让模型不只沿着一条思路推下去，而是像搜索树一样探索多条可能路径。**
