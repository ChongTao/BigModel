# 一 Plan-and-Execute

Plan-and-Execute 是一种把“规划”和“执行”明确拆开的推理方式。

它的核心思路是：

- 先由 Planner 生成计划
- 再由 Executor 按计划逐步执行

## 1.1 它解决什么问题

有些复杂任务如果直接让模型一边想一边做，容易出现：

- 计划不完整
- 中途偏航
- 执行阶段和规划阶段混在一起

Plan-and-Execute 的目标，就是把这两个阶段解耦：

- 规划阶段专注于“先把路想清楚”
- 执行阶段专注于“按步骤落实”

## 1.2 两个核心角色

### 1.2.1 Planner

Planner 负责生成详细任务步骤，例如：

- 先查什么
- 再做什么
- 每一步需要什么输入

### 1.2.2 Executor

Executor 负责按照既定计划逐项执行。

这样做的好处是：执行器不必承担太复杂的全局规划负担，更容易提高稳定性。

## 1.3 适用场景

- 多阶段任务
- 软件开发 Agent
- 自动化工作流
- 复杂规划与执行场景

## 1.4 优点

- 规划和执行职责清晰
- 更容易维护和调试
- 对复杂流程任务更可控

## 1.5 局限

- 对 Planner 质量要求高
- 计划一旦差，后续执行也会被带偏
- 不一定适合非常即时、一步响应的任务

## 1.7 一句话总结

Plan-and-Execute 的核心价值是：**把“想清楚怎么做”和“真正去做”拆成两个阶段，从而提升复杂任务的可控性。**
