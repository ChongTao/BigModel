# 一 ACP 介绍

ACP（Agent Connect Protocol）是一类面向 Agent 调用与互操作的开放协议，重点解决的是：

- 如何标准化描述一个远程 Agent
- 如何通过统一接口调用远程 Agent
- 如何约定输入、输出、状态与控制行为

从定位上看，ACP 和 A2A 属于同一层的问题域，都是 **Agent 互操作协议**，但强调点不同：

- **A2A** 更强调 agent-to-agent 协作、任务、消息、流式更新
- **ACP** 更强调通过标准 API 暴露和调用 Agent 能力

## 1.1 ACP 在目录中的位置

把 ACP 放在 `Agent互操作协议` 目录下，而不是放到“具体框架实现”，是因为它首先是一个**协议层标准**，而不是某个特定 SDK 或框架。

ACP 更适合和 A2A 并列理解：

- 它们都在解决 Agent 如何互通
- 只是抽象重点和接口表达方式不同

## 1.2 ACP 关注什么

ACP 主要关注的是远程 Agent 的标准化调用接口，包括：

- Agent 的元信息描述
- 输入输出模式
- 调用方式
- 结果结构
- 状态与错误表达

因此，它更像是“**如何把一个 Agent 暴露成可调用能力**”的协议层抽象。

## 1.3 ACP 与 A2A 的关系

可以把两者理解成同层但侧重点不同的方案：

- **A2A** 更适合强调任务、消息、协作、流式更新和多 Agent 交互
- **ACP** 更适合强调标准化 agent invocation 和接口描述

它们不是简单替代关系，而更像不同设计取向下的互操作协议。

## 1.4 适合关注 ACP 的场景

ACP 更适合以下场景：

- 希望把 Agent 以标准 API 方式暴露给外部系统
- 希望统一不同 Agent 服务的调用接口
- 希望在平台层做 Agent 注册、编排或接入治理
