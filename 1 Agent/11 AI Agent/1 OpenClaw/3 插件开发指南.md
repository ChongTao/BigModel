## 插件系统概览

OpenClaw 的插件系统是一套完整的**能力扩展机制**，允许开发者以模块化的方式扩展 OpenClaw 的功能，包括：

- **新的消息渠道**（Channel Plugin）
- **新的 AI 模型提供商**（Provider Plugin）
- **新的记忆存储后端**（Memory Plugin）
- **新的工具和技能**（Tool Plugin）
- **生命周期钩子**（Hook Plugin）

### 插件的核心优势

| 优势 | 说明 |
|------|------|
| **模块化** | 独立开发、版本管理、按需安装 |
| **标准化** | Plugin SDK 提供统一的开发接口 |
| **易于分发** | 通过 npm 或 ClawHub 发布，支持一键安装 |
| **热更新** | 支持动态加载和卸载（部分场景） |
| **权限隔离** | 通过 PluginRuntime 控制插件的能力边界 |

## 插件类型

OpenClaw 支持 5 种插件类型：

### 1. Channel Plugin（渠道插件）

**作用**：接入新的消息平台（如自建聊天系统、客服平台等）

**能力契约（ChannelCapabilities）**：

| 能力 | 说明 | 必需 |
|------|------|------|
| `Auth` | 账号认证与绑定（OAuth、Token、扫码等） | ✅ |
| `Config` | 渠道级配置 Schema 及 UI 提示 | ✅ |
| `Messaging` | 接收入站消息，解析消息格式 | ✅ |
| `Outbound` | 发送出站消息（文本、图片、文件等） | ✅ |
| `Threading` | 消息线程/话题管理（如 Slack 线程回复） | ❌ |
| `Groups` | 群组/频道管理 | ❌ |
| `Directory` | 用户目录查询 | ❌ |
| `Security` | 渠道级安全策略（DM 白名单、群组权限等） | ❌ |
| `Status` | 渠道状态上报 | ❌ |
| `Streaming` | 流式输出支持 | ❌ |

**示例渠道**：WhatsApp、Telegram、Slack、Discord、飞书、微信等（29 个已支持）

### 2. Provider Plugin（模型提供商插件）

**作用**：接入新的 AI 模型或推理服务

**支持的提供商**：
- Anthropic（Claude）
- DeepSeek（中国）
- OpenAI（GPT-4）
- Google Gemini
- Qwen（阿里）
- Ollama（本地模型）
- Mistral
- xAI
- 等 18+ 供应商

### 3. Memory Plugin（记忆插件）

**作用**：替换默认的记忆存储后端

**可选后端**：
- `memory-core`（默认，基于 SQLite）
- `memory-lancedb`（可选，高性能向量数据库）

> **注意**：Memory 是**独占插槽**，同一时刻只能激活一个记忆插件。

### 4. Tool Plugin（工具插件）

**作用**：为 Agent 添加新的原子工具（如读写文件、调用 API 等）

**内置工具**（7 个）：
- `message-tool` - 发送消息
- `memory-tool` - 搜索记忆
- `browser-tool` - 浏览器自动化
- `canvas-tool` - UI 绘制
- `cron-tool` - 定时任务
- `gateway-tool` - Gateway 内部调用
- `discord-actions` - Discord 特定操作

### 5. Hook Plugin（钩子插件）

**作用**：在特定生命周期事件注入自定义逻辑

**常见钩子**：
- `message:before` - 消息预处理
- `message:after` - 消息后处理
- `agent:start` - Agent 启动
- `agent:stop` - Agent 停止
- `audit:log` - 审计日志

## 开发插件基础

### 最小插件示例

```typescript
// src/index.ts
import { definePlugin } from '@openclaw/plugin-sdk'

export default definePlugin({
  id: 'my-plugin',
  setup(runtime: PluginRuntime) {
    console.log('Plugin initialized!')
  }
})
```

### 访问 PluginRuntime

`PluginRuntime` 是插件的运行时上下文，提供所有必要的 API：

```typescript
interface PluginRuntime {
  // 基础信息
  version: string                      // OpenClaw 版本
  config: RuntimeConfig                // 配置访问

  // 子功能模块
  subagent: SubagentRuntime           // 子 Agent 调度
  system: SystemRuntime               // 系统操作（文件、进程等）
  media: MediaRuntime                 // 媒体处理（图片、视频等）
  tts: { textToSpeechTelephony }      // 文本转语音
  tools: ToolsRuntime                 // 工具执行
  channel: ChannelRuntime             // 渠道操作
  events: EventsRuntime               // 事件总线
  logging: LoggingRuntime             // 日志（自动脱敏）
  modelAuth: {                        // 模型认证
    getApiKeyForModel,
    resolveApiKeyForProvider
  }
}
```

### 获取配置

```typescript
import { definePlugin } from '@openclaw/plugin-sdk'

export default definePlugin({
  id: 'my-plugin',
  setup(runtime) {
    // 获取插件配置
    const apiKey = runtime.config.get('apiKey')
    const endpoint = runtime.config.get('endpoint', 'https://api.example.com')

    // 监听配置变更
    runtime.config.onChange('apiKey', (newKey) => {
      console.log('API Key changed:', newKey)
    })
  }
})
```

### 使用日志

```typescript
import { definePlugin } from '@openclaw/plugin-sdk'

export default definePlugin({
  id: 'my-plugin',
  setup(runtime) {
    const logger = runtime.logging.createLogger('my-plugin')

    logger.debug('Debug message')      // 详细日志
    logger.info('Info message')        // 信息
    logger.warn('Warning message')     // 警告
    logger.error('Error message')      // 错误

    // 自动脱敏敏感信息（API Key、密码等）
    logger.info(`Connected to ${apiKey}`)  // 日志中自动脱敏
  }
})
```

## Plugin SDK API

### Tools Runtime（工具注册）

```typescript
// 注册新工具
runtime.tools.register({
  name: 'my_tool',
  description: '这是我的工具',
  inputSchema: {
    type: 'object',
    properties: {
      query: { type: 'string', description: '搜索查询' }
    },
    required: ['query']
  },
  run: async (input: { query: string }) => {
    // 工具实现
    return { result: `Found: ${input.query}` }
  }
})

// 工具错误处理
class ToolInputError extends Error {
  status = 400  // 参数错误
}

class ToolAuthorizationError extends ToolInputError {
  status = 403  // 权限不足
}
```

### Channel Runtime（渠道操作）

```typescript
// 发送消息到特定渠道
await runtime.channel.send({
  channelId: 'my-channel',
  message: {
    type: 'text',
    content: 'Hello, World!'
  }
})

// 获取渠道状态
const status = await runtime.channel.getStatus('telegram')

// 监听渠道事件
runtime.channel.on('message', async (event) => {
  console.log('New message:', event)
})
```

### Events Runtime（事件总线）

```typescript
// 发布事件
runtime.events.publish('my-event', { data: 'something' })

// 订阅事件
runtime.events.subscribe('gateway:ready', () => {
  console.log('Gateway is ready!')
})

// 注销事件
runtime.events.unsubscribe('my-event')
```

### Media Runtime（媒体处理）

```typescript
// 生成图片缩略图
const thumbnail = await runtime.media.generateThumbnail('image.jpg', {
  width: 200,
  height: 200
})

// 转换图片格式
const webp = await runtime.media.convert('image.png', { format: 'webp' })

// 获取媒体元数据
const metadata = await runtime.media.getMetadata('video.mp4')
```

### System Runtime（系统操作）

```typescript
// 读取文件
const content = await runtime.system.readFile('/path/to/file')

// 写入文件
await runtime.system.writeFile('/path/to/file', 'content')

// 执行系统命令
const result = await runtime.system.exec('ls -la /tmp')

// 创建临时目录
const tempDir = await runtime.system.createTempDir()
```

## 不同类型插件的开发

### 开发 Tool Plugin（工具插件）

**示例：GitHub 仓库查询工具**

```typescript
// src/index.ts
import { definePlugin } from '@openclaw/plugin-sdk'

export default definePlugin({
  id: 'github-query',
  setup(runtime) {
    runtime.tools.register({
      name: 'github_search_repos',
      description: '搜索 GitHub 仓库',
      inputSchema: {
        type: 'object',
        properties: {
          query: {
            type: 'string',
            description: '搜索关键词（如"react"）'
          },
          limit: {
            type: 'number',
            description: '返回结果数量（默认10）',
            default: 10
          }
        },
        required: ['query']
      },
      run: async (input) => {
        const apiKey = runtime.config.get('githubToken')

        const response = await fetch(
          `https://api.github.com/search/repositories?q=${input.query}&per_page=${input.limit}`,
          {
            headers: { 'Authorization': `Bearer ${apiKey}` }
          }
        )

        if (!response.ok) {
          throw new Error(`GitHub API error: ${response.statusText}`)
        }

        const data = await response.json()
        return {
          count: data.items.length,
          repos: data.items.map(r => ({
            name: r.name,
            url: r.html_url,
            stars: r.stargazers_count,
            description: r.description
          }))
        }
      }
    })
  }
})
```

### 开发 Channel Plugin（渠道插件）

**示例：自定义 HTTP 渠道**

```typescript
// src/index.ts
import { definePlugin } from '@openclaw/plugin-sdk'
import express from 'express'

export default definePlugin({
  id: 'http-channel',
  setup(runtime) {
    const app = express()
    app.use(express.json())

    // 处理入站消息
    app.post('/webhook', async (req, res) => {
      const { userId, message } = req.body

      // 转发给 Agent
      await runtime.channel.send({
        channelId: 'http-channel',
        userId: userId,
        message: {
          type: 'text',
          content: message
        }
      })

      res.json({ status: 'ok' })
    })

    // 启动 HTTP 服务
    const port = runtime.config.get('port', 3001)
    app.listen(port, () => {
      runtime.logging.info(`HTTP Channel listening on port ${port}`)
    })
  }
})
```

### 开发 Provider Plugin（模型提供商插件）

**示例：接入自定义 LLM API**

```typescript
// src/index.ts
import { definePlugin } from '@openclaw/plugin-sdk'

export default definePlugin({
  id: 'my-llm-provider',
  setup(runtime) {
    // 注册新模型提供商
    runtime.tools.register({
      name: 'call_my_llm',
      description: '调用自定义 LLM',
      inputSchema: {
        type: 'object',
        properties: {
          prompt: { type: 'string' },
          temperature: { type: 'number', default: 0.7 }
        },
        required: ['prompt']
      },
      run: async (input) => {
        const apiEndpoint = runtime.config.get('apiEndpoint')
        const apiKey = runtime.modelAuth.getApiKeyForModel('my-llm')

        const response = await fetch(`${apiEndpoint}/completions`, {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${apiKey}`,
            'Content-Type': 'application/json'
          },
          body: JSON.stringify({
            prompt: input.prompt,
            temperature: input.temperature,
            maxTokens: 1000
          })
        })

        const data = await response.json()
        return { text: data.result }
      }
    })
  }
})
```

      plugin.setup(runtime)
    }).not.toThrow()

  })

## 常见问题

### Q: 插件可以访问用户的私密数据吗？

**A:** 不可以。PluginRuntime 通过权限模型限制插件的能力。插件只能访问明确授予的资源（如指定的工具、渠道）。

### Q: 如何在插件中调用其他插件的功能？

**A:** 通过事件总线（Events Runtime）发送事件，其他插件可以监听和响应。

```typescript
// 发布事件
runtime.events.publish('my-event', { data: 'value' })

// 其他插件监听
runtime.events.subscribe('my-event', (data) => {
  console.log('Received:', data)
})
```

### Q: 插件什么时候被加载？

**A:** Gateway 启动时自动发现并加载所有已安装的插件。可以在 `~/.openclaw/extensions/` 目录中放置本地开发的插件。

### Q: 如何调试插件中的异步问题？

**A:** 使用日志记录关键步骤，启用 OpenClaw 的调试模式：

```bash
openclaw --debug gateway start
```

### Q: 插件可以修改 Agent 的行为吗？

**A:** 可以，通过 Hook Plugin 在关键生命周期注入逻辑。但不建议直接修改 Agent 核心，应通过官方扩展点进行。

### Q: 如何处理插件的依赖冲突？

**A:** 在 package.json 中明确指定依赖版本范围，并在发布前测试与其他常用插件的兼容性。
