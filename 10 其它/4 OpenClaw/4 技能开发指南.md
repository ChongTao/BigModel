## 技能系统概览

OpenClaw **Skills**（技能）是一套**业务级的能力封装**，由一个或多个工具（Tools）和规则组成，为用户提供完整的功能场景。

### 技能 vs 工具

| 维度 | Skills（技能） | Tools（工具） |
|------|----------------|--------------|
| **粒度** | 业务级（如"管理 GitHub Issues"） | 原子级（如"读取文件"） |
| **来源** | 独立包，可单独安装/卸载 | 内置于 Agent 核心 |
| **发现方式** | `AGENTS.md` 注册 | `TOOLS.md` 注册 |
| **分发** | ClawHub 技能市场 | 随 OpenClaw 核心发布 |
| **示例** | `@openclaw/github-issues`、`@openclaw/notion` | `message-tool`、`memory-tool` |

### 技能的特点

```
用户需求
  ↓
技能理解意图，拆解成子任务
  ↓
调用多个工具完成各子任务
  ↓
综合结果返回给用户
```

**例子：GitHub Issues 技能**
- 用户说："创建一个新的 GitHub Issue"
- 技能接收意图，调用：
  1. `github-auth-tool` - 验证 GitHub 凭证
  2. `github-create-issue-tool` - 创建 Issue
  3. `message-tool` - 反馈创建结果

```

├── src/
```

### 步骤 2：实现技能主体

**src/index.ts**

```typescript
import { defineSkill } from '@openclaw/plugin-sdk'
import { createIssueHandler } from './handlers/create'
import { listIssuesHandler } from './handlers/list'
import { closeIssueHandler } from './handlers/close'

export default defineSkill({
  id: 'github-issues',
  displayName: 'GitHub Issues Manager',
  description: 'Manage GitHub Issues',

  // 配置初始化
  async initialize(runtime) {
    const token = runtime.config.get('github_token')
    if (!token) {
      throw new Error('GitHub token is required. Please configure it.')
    }

    const logger = runtime.logging.createLogger('github-issues')
    logger.info('GitHub Issues skill initialized')
  },

  // 定义意图处理
  handlers: {
    create_issue: createIssueHandler,
    list_issues: listIssuesHandler,
    close_issue: closeIssueHandler
  },

  // 设置权限检查（可选）
  async authorize(runtime, context) {
    const userPerms = context.userPermissions || []
    return userPerms.includes('github:write')
  }
})
```

### 步骤 3：实现处理器

**src/handlers/create.ts**

```typescript
import { Octokit } from '@octokit/rest'

export async function createIssueHandler(input, runtime) {
  const token = runtime.config.get('github_token')
  const octokit = new Octokit({ auth: token })

  const logger = runtime.logging.createLogger('github-issues:create')

  try {
    logger.info(`Creating issue: ${input.title}`)

    const response = await octokit.issues.create({
      owner: input.owner,
      repo: input.repo,
      title: input.title,
      body: input.description,
      labels: input.labels || [],
      assignees: input.assignees || []
    })

    logger.info(`Issue created: ${response.data.number}`)

    return {
      success: true,
      issueNumber: response.data.number,
      issueUrl: response.data.html_url,
      message: `✅ Issue #${response.data.number} created successfully`
    }
  } catch (error) {
    logger.error(`Failed to create issue: ${error.message}`)

    return {
      success: false,
      error: error.message,
      message: `❌ Failed to create issue: ${error.message}`
    }
  }
}
```

**src/handlers/list.ts**

```typescript
import { Octokit } from '@octokit/rest'

export async function listIssuesHandler(input, runtime) {
  const token = runtime.config.get('github_token')
  const octokit = new Octokit({ auth: token })

  const logger = runtime.logging.createLogger('github-issues:list')

  try {
    logger.info(`Listing issues for ${input.owner}/${input.repo}`)

    const response = await octokit.issues.listForRepo({
      owner: input.owner,
      repo: input.repo,
      state: input.state || 'open',
      per_page: input.limit || 10
    })

    const issues = response.data.map(issue => ({
      number: issue.number,
      title: issue.title,
      state: issue.state,
      url: issue.html_url,
      createdAt: issue.created_at,
      updatedAt: issue.updated_at
    }))

    return {
      success: true,
      count: issues.length,
      issues,
      message: `Found ${issues.length} issues`
    }
  } catch (error) {
    logger.error(`Failed to list issues: ${error.message}`)

    return {
      success: false,
      error: error.message,
      message: `❌ Failed to list issues: ${error.message}`
    }
  }
}
```

**src/handlers/close.ts**

```typescript
import { Octokit } from '@octokit/rest'

export async function closeIssueHandler(input, runtime) {
  const token = runtime.config.get('github_token')
  const octokit = new Octokit({ auth: token })

  const logger = runtime.logging.createLogger('github-issues:close')

  try {
    logger.info(`Closing issue #${input.issueNumber}`)

    const response = await octokit.issues.update({
      owner: input.owner,
      repo: input.repo,
      issue_number: input.issueNumber,
      state: 'closed'
    })

    logger.info(`Issue #${response.data.number} closed`)

    return {
      success: true,
      issueNumber: response.data.number,
      message: `✅ Issue #${response.data.number} closed successfully`
    }
  } catch (error) {
    logger.error(`Failed to close issue: ${error.message}`)

    return {
      success: false,
      error: error.message,
      message: `❌ Failed to close issue: ${error.message}`
    }
  }
}
```

## 常见问题

### Q: 如何让 Agent 自动选择我的技能？

**A:** 在 skill.manifest.json 中编写清晰的 `examples`，Agent 会根据用户的自然语言意图匹配最相关的技能。

### Q: 技能和工具有什么区别？

**A:** 技能是高级业务功能（由一个或多个工具组成），工具是低级原子操作（内置于 Agent 核心）。

### Q: 可以在技能中调用其他技能吗？

**A:** 技能是独立的单位，应避免技能间依赖。如果需要组合功能，应在 Agent 层面进行编排。

### Q: 如何处理认证信息？

**A:** 通过 `runtime.config.get()` 从配置获取敏感信息，不要硬编码 API Key 或密码。

### Q: 技能支持流式响应吗？

**A:** 是的，可以通过 `runtime.channel.sendStream()` 实现流式输出。

### Q: 如何调试技能中的异步问题？

**A:** 使用 logger 在关键步骤记录，并启用 OpenClaw 的调试模式：

```bash
openclaw --debug gateway start
```

### Q: 我可以发布私有技能吗？

**A:** 可以，将技能发布到私有 npm 仓库或放在 `~/.openclaw/extensions/` 目录中本地加载。
