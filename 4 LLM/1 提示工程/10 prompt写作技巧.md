良好设计的 Prompt 通常由任务说明、上下文、问题、输出格式 四个基本元素组成：

- 任务说明：是 Prompt 的核心，用来明确模型的角色、目标和要完成的任务，避免模型“自由发挥”过头。
  - 常见形式：你需要做什么（分析 / 总结 / 翻译 / 生成 / 对比等）
  - 示例：你是一名资深产品经理，请帮助我分析以下产品需求。

- 上下文：帮助模型理解任务发生的环境、限制条件和已有信息，相当于“背景资料”。
  - 常见内容：业务背景、目标用户、已知用户

  - 示例：背景：该产品是一款面向大学生的时间管理 App，目前用户留存率较低。
- 问题： Prompt 中“最直接的输入”，明确模型需要思考和输出的核心点。
  - 常见形式：需要分析什么、需要回答哪些点。

  - 示例：请分析导致用户留存率低的 3 个主要原因。
- 输出格式：能显著提升结果的**可读性、可复用性和可控性**，尤其在工作场景中非常重要。
  - 常见格式：列表、表格、Markdown。
  - 示例：使用 Markdown形式输出。

```json
任务说明：
你是一名资深产品经理，请从专业角度进行分析。

上下文：
该产品是一款面向大学生的时间管理 App，目前用户留存率较低。

问题：
请分析用户留存率低的 3 个主要原因。

输出格式：
请使用 Markdown 编号列表，每点不超过 100 字。
```
