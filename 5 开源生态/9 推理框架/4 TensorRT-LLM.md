# 4 TensorRT-LLM

`TensorRT-LLM` 是 NVIDIA 生态里的高性能大模型推理框架，核心定位是：

> 在 NVIDIA GPU 环境下，把大模型推理做得更快、更省、更接近生产级极限性能。

如果说 `TGI` 更偏标准化服务，`vLLM` 更偏通用高并发推理，那么 `TensorRT-LLM` 更偏 **底层性能优化、低精度推理和 NVIDIA 平台深度适配**。

## 4.1 它解决什么问题

很多服务端推理系统的瓶颈，不只是“能不能跑”，而是：

- 单卡吞吐不够高
- 延迟压不下来
- 大模型多卡部署成本高
- 低精度推理没有吃满硬件能力
- GPU 资源利用率离理论上限还很远

`TensorRT-LLM` 的目标就是尽量把这些性能瓶颈继续往下压。

## 4.2 核心特点

### 4.2.1 深度依赖 NVIDIA 生态

`TensorRT-LLM` 的一个核心特点，是它不是通用抽象层，而是明显建立在 NVIDIA 软件栈之上：

- 与 CUDA 生态结合深
- 与 TensorRT 优化能力结合深
- 更适合 NVIDIA GPU 集群和生产环境

这意味着它通常不是“最通用”的方案，但在 NVIDIA 环境里往往能换来更强性能。

### 4.2.2 更强调底层推理优化

它的强项不只是“把模型封装成服务”，而是通过更深的底层优化去逼近性能上限，例如：

- kernel fusion
- 图优化
- 更激进的执行路径优化
- 更细粒度地利用 GPU 计算能力

所以它更像“面向性能极限的推理引擎”，而不是“上手最简单的服务框架”。

### 4.2.3 低精度与多卡能力更重要

`TensorRT-LLM` 经常和这些关键词一起出现：

- INT8
- INT4
- FP8
- Tensor Parallel
- Pipeline Parallel

这说明它非常适合：

- 大模型低精度推理
- 高性能多 GPU 部署
- 需要把吞吐、延迟和成本一起优化的场景

## 4.3 系统里它处在什么位置

和 `vLLM`、`TGI` 一样，`TensorRT-LLM` 位于模型执行层：

```text
应用 / Agent / RAG
   ↓
API 网关 / 统一接入层
   ↓
TensorRT-LLM
   ↓
NVIDIA GPU
   ↓
模型权重
```

如果放到仓库现有概念里：

- `OneAPI` 负责统一接入、配额、路由和治理
- `TensorRT-LLM` 负责把模型在 NVIDIA 平台上高性能跑起来
- `Harness` 负责更外围的运行控制环境

## 4.4 它适合什么场景

`TensorRT-LLM` 比较适合：

- 已经确定使用 NVIDIA GPU 作为核心算力平台
- 对吞吐、延迟和成本都要求很高
- 需要多卡部署大模型
- 愿意为了性能收益接受更高工程复杂度

如果你更看重部署标准化和快速落地，`TGI` 往往更直接；如果你更看重高并发通用在线推理，很多场景会优先考虑 `vLLM`；如果你更看重 NVIDIA 环境下的性能逼近，`TensorRT-LLM` 更值得优先评估。

## 4.5 与 vLLM、TGI 的区别

| 对比项 | TensorRT-LLM | vLLM | TGI |
| :--- | :--- | :--- | :--- |
| 核心定位 | NVIDIA 高性能推理框架 | 通用高并发在线推理框架 | 标准化推理服务 |
| 强项 | 底层性能优化、低精度、多卡 | KV Cache 管理、连续批处理 | Hugging Face 生态、服务化部署 |
| 环境依赖 | NVIDIA 生态强依赖 | 相对更通用 | 相对更偏服务交付 |
| 典型选择场景 | 极致性能生产环境 | 通用在线推理服务 | 快速标准化部署 |

简单理解：

- `vLLM` 更像“高并发在线推理优先”
- `TGI` 更像“标准服务部署优先”
- `TensorRT-LLM` 更像“性能极限优先”

## 4.6 与 Ollama、OneAPI 的区别

| 名称 | 主要定位 |
| :--- | :--- |
| TensorRT-LLM | 推理框架 / 高性能执行引擎 |
| Ollama | 本地运行时与模型管理产品 |
| OneAPI | 模型网关与统一接入层 |

所以：

- `Ollama` 解决的是本地模型运行和管理
- `OneAPI` 解决的是多模型服务统一接入
- `TensorRT-LLM` 解决的是 NVIDIA 平台上的高性能推理执行

## 4.7 优势与限制

优势：

- 更容易逼近 NVIDIA 平台的推理性能上限
- 低精度推理能力重要
- 多卡和大模型生产环境更有优势

限制：

- 明显依赖 NVIDIA 生态
- 工程复杂度通常高于“开箱即用型”服务框架
- 不一定是本地开发或通用部署的第一选择

## 4.8 一句话总结

`TensorRT-LLM` 的核心价值，在于把大模型推理深度贴合到 NVIDIA 硬件和软件栈上，以换取更强的性能、低精度能力和生产级多卡部署能力。
