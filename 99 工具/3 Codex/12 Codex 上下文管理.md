# 12 Codex 上下文管理

Codex 的稳定性，很大程度取决于上下文怎么组织，而不是提示词写得多华丽。

在 Codex 里，常见上下文来源包括：

- 当前提示词
- 当前目录和文件
- `AGENTS.md`
- Memories
- Skills
- MCP 提供的外部上下文
- Subagents 的独立线程上下文

## 12.1 为什么要管上下文

复杂工程任务里最常见的问题不是“模型不会”，而是：

- 上下文太杂
- 任务目标不稳定
- 无关信息挤占窗口
- 规则和一次性需求混在一起

## 12.2 一次性信息 vs 长期信息

建议这样分：

- 一次性任务说明：放在当前 prompt
- 长期项目规则：放在 `AGENTS.md`
- 可复用流程：放在 Skill
- 有用的历史偏好：交给 Memory

不要把所有东西都塞进一次 prompt。

## 12.3 记忆不是规则替代品

Codex 的 Memories 更适合保存：

- 稳定偏好
- 常用技术栈
- 重复出现的工作习惯
- 已知坑点

但必须强制生效的规则，还是应该写进：

- `AGENTS.md`
- 仓库文档
- Skill

## 12.4 管理上下文的实用方法

- 让仓库规则尽量近目录放置
- 把重复流程沉淀进 Skill
- 把外部资料通过 MCP 接入
- 复杂任务显式拆子代理，减少单线程污染
- 改完一次规则后，顺手更新 `AGENTS.md`
