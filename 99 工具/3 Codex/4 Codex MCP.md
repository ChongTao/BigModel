# 4 Codex MCP

`MCP` 是 Codex 接入外部工具和上下文的重要方式之一。

如果只看作用，它解决的是：

> 让 Codex 能访问第三方文档、系统和工具，而不只是局限在本地文件与终端。

## 4.1 Codex 里的 MCP 是什么

在 Codex 里，MCP 常见用途包括：

- 接文档系统
- 接浏览器或设计工具
- 接内部平台
- 接开发辅助服务

它更像“给 Codex 增加工具与上下文入口”的协议层。

## 4.2 适合什么场景

MCP 很适合：

- 让 Codex 读第三方文档
- 让 Codex 操作外部系统
- 让团队共享统一工具接入方式
- 把工具能力和本地工程协作串起来

## 4.3 和 Plugin、Skill 的区别

| 能力 | 更接近什么 |
| :--- | :--- |
| MCP | 工具接入协议 |
| Skill | 工作流与方法封装 |
| Plugin | 可安装分发包 |

简单理解：

- Skill 负责“怎么做”
- MCP 负责“能接到什么工具”
- Plugin 负责“怎么打包给别人装”

## 4.4 在 Codex 里的位置

Codex 的常见自定义层可以这样理解：

- `AGENTS.md`：规则
- `Memories`：记忆
- `Skills`：可复用工作流
- `MCP`：外部工具与上下文
- `Subagents`：并行代理

MCP 不替代其他层，而是补足“外部能力接入”。
