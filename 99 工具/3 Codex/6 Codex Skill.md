# 6 Codex Skill

在 Codex 里，`Skill` 是可复用的工作流封装。

可以把它理解成：

> 把一类任务的做法、指令、资源和可选脚本封装起来，让 Codex 以后遇到类似任务时能按固定方法执行。

## 6.1 Skill 的作用

Skill 适合封装：

- 固定排查流程
- 固定 review 流程
- 特定技术栈操作手册
- 文档生成流程
- 团队内部约定动作

## 6.2 Skill 和 Plugin 的关系

这两者很容易混淆，但不一样：

| 概念 | 作用 |
| :--- | :--- |
| Skill | 工作流作者格式 |
| Plugin | 安装与分发单位 |

简单理解：

- Skill 解决“工作流怎么写”
- Plugin 解决“这个工作流怎么安装给别人”

## 6.3 Skill 适合什么场景

如果你还在单仓库内反复打磨一个流程，优先用 Skill。

例如：

- 安全 review 技能
- 文档重构技能
- Java 服务排障技能
- SQL 变更审查技能

## 6.4 Skill 的典型组成

一个 Skill 常见会包含：

- `SKILL.md`
- 说明描述
- 触发条件
- 工作步骤
- 参考资源
- 可选脚本

## 6.5 什么时候不该先做 Skill

如果流程还没稳定，就不要急着封装成 Skill。先把任务多做几次，确定流程稳定，再沉淀。
