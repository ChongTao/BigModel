# 7 Codex Plugin

在 Codex 里，`Plugin` 是可安装、可分发的扩展包。

它通常会把这些东西打包在一起：

- Skills
- App 集成
- MCP 配置
- 生命周期钩子

## 7.1 Plugin 的定位

Plugin 更像“交付单元”，而不是单纯的工作流定义。

如果 Skill 是“做法”，那么 Plugin 更像“安装包”。

## 7.2 适合什么场景

Plugin 更适合：

- 团队内部共享能力
- 跨项目复用
- 把 Skills 和 MCP 一起打包
- 对外分发稳定能力

## 7.3 和 MCP、Skill 的边界

| 能力 | 主要作用 |
| :--- | :--- |
| Skill | 定义工作流 |
| MCP | 接入外部工具 |
| Plugin | 打包安装与分发 |

## 7.4 实际理解

你可以把一套能力链路理解成：

1. 先写 Skill
2. 再决定是否接 MCP
3. 最后打成 Plugin 给团队安装

## 7.5 不要机械上 Plugin

如果只是个人本地仓库里反复使用一个流程，通常先做 Skill 就够了。  
只有当你要共享、分发、标准化管理时，再考虑 Plugin。
