# 8 Codex Subagents

`Subagents` 是 Codex 处理复杂任务时的并行化能力。

它的核心思想是：

> 把大任务拆成多个可并行的子任务，交给多个专门代理分别处理，再由 Codex 汇总结果。

## 8.1 什么时候用 Subagents

Subagents 特别适合：

- 大仓库并行调研
- 多维度代码审查
- 多模块独立分析
- 复杂任务拆解

例如：

- 一个代理看安全
- 一个代理看 bug
- 一个代理看测试风险
- 一个代理看可维护性

## 8.2 它的价值

即使上下文窗口很大，单线程代理仍然容易出现：

- 上下文污染
- 注意力分散
- 长链路任务混乱

Subagents 的价值就是把这些问题分摊掉。

## 8.3 使用时要注意

- 子代理不是默认乱开，最好显式要求
- 子代理会增加 token 消耗
- 子代理适合并行，不适合强顺序依赖任务
- 子代理也继承当前的权限和沙箱策略

## 8.4 常见用法

```text
对当前分支做 review。请开 4 个子代理，分别检查：
1. bug
2. 安全风险
3. 性能问题
4. 测试缺口
等全部完成后统一汇总。
```

## 8.5 自定义 agents

Codex 还支持自定义 agent 角色。你可以为不同类型的任务准备不同代理，例如：

- explorer：偏阅读分析
- worker：偏执行实现
- reviewer：偏审查总结

这比所有任务都塞给一个默认代理更稳定。
